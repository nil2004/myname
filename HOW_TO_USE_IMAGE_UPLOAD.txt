================================================================================
HOW TO USE THE IMAGE UPLOAD SYSTEM
================================================================================

STEP 1: RUN THE DATABASE MIGRATION
-----------------------------------
1. Open Supabase Dashboard: https://uaiwuivyrdoausenvlbs.supabase.co
2. Go to SQL Editor
3. Copy and paste the contents of: add-vendor-media-columns.sql
4. Click "Run" to execute the migration
5. Verify success by checking the vendors table structure

STEP 2: CREATE STORAGE BUCKET (if not exists)
----------------------------------------------
The SQL script above creates the bucket automatically, but if needed:
1. Go to Supabase Dashboard → Storage
2. Create new bucket named: "vendor-media"
3. Make it PUBLIC
4. Set policies for authenticated uploads

STEP 3: UPLOAD IMAGES FROM ADMIN PANEL
---------------------------------------
1. Go to your admin panel: http://localhost:3000/admin
2. Click on "Vendors Management" tab
3. Click "Add Vendor" or "Edit" on existing vendor
4. In the modal, scroll to "Portfolio & Media" section
5. Upload images:
   
   📸 BANNER IMAGE (Main Featured Image):
   - Click "Choose File" under Banner Image
   - Select ONE high-quality image
   - This will be the main vendor image shown to customers
   - Preview appears immediately
   - Click X to remove if needed
   
   📷 PORTFOLIO IMAGES (Multiple):
   - Click "Choose File" under Portfolio Images
   - Select MULTIPLE images (Ctrl/Cmd + Click)
   - These show in vendor portfolio gallery
   - Each image has a remove button
   
   🎥 PORTFOLIO VIDEOS (Optional):
   - Click "Choose File" under Portfolio Videos
   - Select MULTIPLE videos (Ctrl/Cmd + Click)
   - Videos show with player controls
   - Each video has a remove button

6. Fill in other vendor details
7. Click "Create Vendor" or "Update Vendor"
8. Images are saved to database automatically!

STEP 4: VIEW IMAGES ON CUSTOMER SITE
-------------------------------------
1. Go to: http://localhost:3000/plan
2. Fill in the planning form
3. Click "Get AI Recommendations"
4. You'll see vendors with their uploaded images!
5. If no image uploaded, emoji fallback shows

================================================================================
FILE UPLOAD SPECIFICATIONS
================================================================================

SUPPORTED FORMATS:
- Images: JPG, PNG, GIF, WebP, etc. (any image/*)
- Videos: MP4, WebM, MOV, etc. (any video/*)

FILE SIZE LIMITS:
- Maximum: 50MB per file
- Recommended: 2-5MB for images, 10-20MB for videos

IMAGE RECOMMENDATIONS:
- Banner Image: 1200x600px or 16:9 ratio
- Portfolio Images: 800x600px or 4:3 ratio
- Format: JPG or PNG
- Quality: High (80-90%)

VIDEO RECOMMENDATIONS:
- Resolution: 1080p or 720p
- Format: MP4 (H.264)
- Duration: 30-60 seconds
- Size: Under 20MB

================================================================================
TECHNICAL DETAILS
================================================================================

API ENDPOINTS:
--------------
POST   /api/vendors/upload    - Upload single file (image/video)
POST   /api/vendors           - Create new vendor
PUT    /api/vendors           - Update existing vendor
DELETE /api/vendors?id=xxx    - Delete vendor
DELETE /api/vendors/upload?fileName=xxx - Delete uploaded file

STORAGE LOCATION:
-----------------
Bucket: vendor-media
URL Pattern: https://uaiwuivyrdoausenvlbs.supabase.co/storage/v1/object/public/vendor-media/{filename}

DATABASE FIELDS:
----------------
- banner_image: TEXT (single URL)
- portfolio_images: TEXT[] (array of URLs)
- portfolio_videos: TEXT[] (array of URLs)
- portfolio_description: TEXT
- portfolio_highlights: TEXT[]

UPLOAD FLOW:
------------
1. User selects file in admin panel
2. File sent to /api/vendors/upload via FormData
3. API validates file type and size
4. File uploaded to Supabase Storage
5. Public URL returned
6. URL stored in component state
7. When user clicks Save, URL saved to database
8. Customer site fetches vendor with image URLs
9. Images displayed in recommendations

================================================================================
TROUBLESHOOTING
================================================================================

PROBLEM: "Upload failed: Bucket not found"
SOLUTION: Run the SQL migration script to create the bucket

PROBLEM: "Upload failed: File too large"
SOLUTION: Compress image/video to under 50MB

PROBLEM: Images not showing on customer site
SOLUTION: 
- Check if banner_image field has URL in database
- Verify URL is accessible (open in browser)
- Check browser console for errors
- Ensure vendor is verified (only verified vendors show)

PROBLEM: Upload button disabled
SOLUTION: Wait for previous upload to complete

PROBLEM: "Failed to upload file"
SOLUTION:
- Check Supabase credentials in .env.local
- Verify storage bucket exists and is public
- Check browser console for detailed error
