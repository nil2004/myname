🧠 HOW THE AI MATCHING WORKS:
═══════════════════════════════════════════════════════════════

The AI calculates a MATCH SCORE for each vendor based on multiple factors:

┌─────────────────────────────────────────────────────────────┐
│  SCORING SYSTEM (Total: 0-24 points)                        │
└─────────────────────────────────────────────────────────────┘

1. RATING SCORE (0-5 points)
   ├── Direct rating value
   └── Example: 4.8 rating = 4.8 points

2. PRICE MATCH SCORE (0-3 points)
   ├── Well within budget (≤80%): 3 points
   ├── Within budget (≤100%): 2 points
   ├── Slightly over (≤120%): 1 point
   └── Too expensive: 0 points

3. THEME MATCH IN DESCRIPTION (0-5 points) ⭐ NEW!
   ├── Searches vendor description for theme keywords
   ├── Each keyword match: +1.5 points
   ├── Maximum: 5 points
   └── Example: "Cartoon" theme finds "kids", "fun", "colorful"

4. THEME MATCH IN TAGS (0-5 points) ⭐ NEW!
   ├── Searches vendor tags for theme keywords
   ├── Each tag match: +2 points
   ├── Maximum: 5 points
   └── Example: Tags ["kids-friendly", "cartoon-themes"]

5. EXPERIENCE BONUS (0-2 points)
   ├── 10+ years: 2 points
   ├── 5-9 years: 1 point
   └── <5 years: 0 points

6. EVENTS DONE BONUS (0-2 points)
   ├── 200+ events: 2 points
   ├── 100-199 events: 1 point
   └── <100 events: 0 points

7. GUEST COUNT SUITABILITY (0-2 points)
   ├── For restaurants with per-plate pricing
   ├── Estimated cost within budget: 2 points
   ├── Estimated cost slightly over: 1 point
   └── Too expensive: 0 points

🔍 KEYWORD MATCHING LOGIC:
═══════════════════════════════════════════════════════════════

Description Matching:
├── Converts description to lowercase
├── Searches for each theme keyword
├── Counts matches
├── Awards 1.5 points per match (max 5 points)
└── Example: "Fun colorful kids party" = 3 matches = 4.5 points

Tag Matching:
├── Converts all tags to lowercase
├── Checks if any tag contains theme keyword
├── Counts matches
├── Awards 2 points per match (max 5 points)
└── Example: ["kids-friendly", "cartoon"] = 2 matches = 4 points

═══════════════════════════════════════════════════════════════
🔄 MATCHING PROCESS FLOW:
═══════════════════════════════════════════════════════════════

1. Customer submits request
   ↓
2. System fetches all verified vendors in city
   ↓
3. Filters by selected categories (Restaurant, Cake, etc.)
   ↓
4. For each vendor, calculates match score:
   ├── Rating score
   ├── Price match score
   ├── Description keyword matches ⭐ NEW!
   ├── Tag keyword matches ⭐ NEW!
   ├── Experience bonus
   ├── Events done bonus
   └── Guest count suitability
   ↓
5. Sorts vendors by score (highest first)
   ↓
6. Selects top vendor from each category
   ↓
7. Returns 3 best-matched vendors
   ↓
8. Admin can review and override if needed
