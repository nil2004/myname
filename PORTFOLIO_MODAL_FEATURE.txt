================================================================================
HOW IT WORKS
================================================================================

1. User clicks "View Portfolio" button on vendor card
2. Modal opens with loading spinner
3. API fetches full vendor details from database
4. Modal displays:
   - Banner image
   - Stats (rating, experience, events)
   - Description
   - Highlights
   - Portfolio images (with gallery)
   - Portfolio videos
   - Pricing information

================================================================================
API ENDPOINT
================================================================================

GET /api/vendors/[id]

Fetches complete vendor data including:
- All basic info (name, category, rating, etc.)
- Banner image
- Portfolio images array
- Portfolio videos array
- Portfolio description
- Portfolio highlights
- Experience and events data
- Pricing information

================================================================================
DATA FLOW
================================================================================

1. VendorTierCard component
   ↓
2. User clicks "View Portfolio"
   ↓
3. Fetch: GET /api/vendors/{vendor.id}
   ↓
4. Supabase query: SELECT * FROM vendors WHERE id = {id}
   ↓
5. Return complete vendor data
   ↓
6. Display in PortfolioModal component

================================================================================
TROUBLESHOOTING
================================================================================

Modal not opening?
- Check browser console for errors
- Verify API endpoint exists
- Check if vendor ID is valid

Images not showing?
- Verify portfolio_images field has URLs
- Check if images are accessible
- Look for CORS errors

Videos not playing?
- Check video format (MP4 recommended)
- Verify video URLs are accessible
- Check file size (should be under 50MB)

API errors?
- Verify Supabase connection
- Check if vendor exists in database
- Look at server logs
